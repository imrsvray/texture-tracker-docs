Detailed Description

Fits a straight line through a given set of points.

Intended for use in applications such as fitting lines through edge points in an image. Includes methods to allow the points and fitted line to be drawn on an image, calculate the intersection between lines, and reject outliers. A RANSAC-like method for robust outlier rejection is also supported.

Graham Thomas, BBC R&D

void bbcvp::FittedLine::calcRange (   ) 
 

Calculate the range of x and y values for the currently-stored points.

Normally this needn't be called by the user, as the range is updated each time a new point is added. However, if you have removed any points using removePoint() or rejectOutliers(), then call this if you want getMinX(), etc., to give results that reflect the reduced point set. Note also that calcRange() only looks through the points that are stored, so if more points have been added than there was space to remember, the range will be incorrect. If no points at all are being stored (i.e. the constructor was called with no argument, or an argument of zero), this function does nothing.

Draw the fitted line on the given image.

The line profile is an inverted V, with the given width between the half-height points, to provide some anti-aliasing. Pixels within the line are drawn as a mixture between lineVal and the value previously in the image, with the mixture depending on the distance formthe line according to the line profile. To draw a line in a multi-component image, call this method separately for each component you want to change. Note that lines with an extent of less than one pixel in their longest direction are not drawn at all. Lines with fewer than 2 points are not drawn either! Only the portion of the line in the rectangle (0,0) to (lenX-1, lenY-1) is drawn.

int bbcvp::FittedLine::getDominantLine (  FittedLine *  dominantLine,
float  maxDistance = 1.0,
float  stopIterFraction = 0.5,
float  stepFraction = 0.0,
float  testFraction = 1.0
) 
 

Find the line that best matches the majority of the stored points using a RANSAC-like technique.

Tests possible lines that pass through various combinations of two of the points, and finds the line that has the largest number of points within a given distance of it. The points within this distance are then placed in the given FittedLine object, leaving the current line untouched. Only operates on the points for which there was sufficient storage, so make sure you used a large enough value for MAX_POINTS when instantiating the FittedLine. The supplied FittedLine object must be different from the current object.

Parameters:
dominantLine Pointer to an existing FittedLine object which will be set to the dominant line.
maxDistance Any points within this distance from a 'test' line will be added to it.
stopIterFraction Stop testing candidate lines when a line that matches at least this fraction of points is found.
stepFraction When picking pairs of points to represent lines to test (in the order they were added to the line), advance by this fraction of the total number of points each time. The spacing between the first and last points to be tested is also decreased by this amount after each pass through. Setting this to 0 means that a step size of 1 will be used, and will result in all combinations of points being tested (=0.5*n*(n-1)). Setting it to 1 means that only one line (from the first to the last point) is tested.
testFraction Test this fraction of the points when determining if a candidate line is any good. Set this to 1 to test all points. Testing fewer points is faster, but may not find the best line.
Returns:
Number of points fitted to dominant line.

bool bbcvp::FittedLine::removePoint (  float  x,
float  y,
bool  recalculateFit = true
) 
 

Remove a point from the line, and optionally recalculate the fitted line.

Note that removing points without recalculating the fit each time is slightly faster, but any calls to methods that use the parameters of the fitted line (such as getEquation, getDistanceToPoint, drawLine, etc.) will not take into account the newly-added points. You must either call calcFit, or addPoint with recalculateFit = true (the default) to take account of all added points. Note that it is possible to remove a point that was never added, as each point only affects the accumulated totals of squared position, etc. If the point was found in the list of added points, it is removed from the list; if it is not found then the last point in the list is removed instead. This does not affect the calculation of the fitted line, but will interfere with the operation of any subsequent outlier rejection. Note also that the range of the points making up the line are not recalculated, so if you remove a point that gave rise to an extreme x and/or y value, calls to getMinX(), etc., will give out-of-date results. If you care about this and don't mind spending some extra computation time, call calcRange() after removing points.

Parameters:
x The x coordinate of the point to add
y The y coordinate of the point to add
recalculateFit Calculate the parameters of the line taking the new point into account
Returns:
true if the point was found in the stored list of points, false if it was not (so a return of false means that the list of stored points is no longer correct)
