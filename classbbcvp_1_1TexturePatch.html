Constructor.

Allocates internal storage for working space.

Parameters:
patchLenX,patchLenYThe dimensions of the patches to store, in pixels/lines (e.g. 16,16)
nResolutionsNumber of levels in the multi-resolution image pyramid used. Defaults to 1 - Indeed, we may not need multi-resolution patches if the frame-to-frame matching (using image pyramids) is used to get us close, and the stored patches are used only for drift prevention. But then again, they could help with dealing with zoom, if we include a method to make a scaled texture patch.....

void bbcvp::TexturePatch::generateWarpedPatch ( double  newFocalLength,
int  lowestLevel,
int  highestLevel 
)

Generate a set of warped image patches for the range of levels requested, to match the given the focal length.

Conceptually it generates a complete image pyramid, but in practice it only computes those levels that are in the given range, leaving the others unfilled.

Call useWarpedPatch(true) to make the warped patch be returned by getImagePointerArray() etc.

)

Copy image data into the given resolution level of the patch.

Parameters:
levelThe level of the image pyramid that the image corresponds to
ImageTopLeftPointer to top-left pixel of image
imageIncYThe vertical increment of the input image (horizontal increment is hard-coded to 1)
lenX,lenYDimensions of the image level (used to stop reading image data outside image if patch falls outside)
centreX,centreYThe pixel at the centre of the patch to extract, in units of pixels at the given level (ie. the given centre coords should be scaled down before calling this) This value is quantised before extracting the patch, and the rounding error noted for use by the getFeatureX/getFeatureY methods.
